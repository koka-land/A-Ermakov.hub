# Архитектура и Шаблоны Экосистемы (Hub)

Документация по ключевым решениям, структуре проекта и готовым шаблонам для быстрого добавления новых инструментов.

---

## 1. Структура проекта (Компонентный подход)

Каждая программа изолирована в своей папке и содержит бэкенд, разметку и персональную логику.
/progs
└── /hello_python        # Папка конкретной программы
├── main.py          # Бэкенд (Python-скрипт)
├── ui.php           # Фронтенд (Только чистая разметка формы)
└── ui.js            # Клиентская логика (Скрипт кнопки запуска)
/tabs                    # Глобальные вкладки интерфейса (Инструменты, Настройки)
/style.css               # Глобальная таблица стилей (Material Design 3)
/index.php               # Главная точка входа
/ui_render.php           # Шлюз для динамической загрузки ui.php программ


---

## 2. Готовый UI-Шаблон для новых программ (`ui.php`)

При создании новой утилиты копируйте этот скелет в `progs/имя_программы/ui.php`. Все стили подтягиваются автоматически из `style.css`.

```html
<div class="program-form-container">
    
    <div class="input-field-container">
        <label for="УНИКАЛЬНЫЙ-ID-ИНПУТА" class="label-medium program-label">
            Понятный текст подписи для пользователя:
        </label>
        <input type="text" id="УНИКАЛЬНЫЙ-ID-ИНПУТА" class="program-input" placeholder="Подсказка...">
    </div>

    <button id="УНИКАЛЬНЫЙ-ID-КНОПКИ" class="ripple-target program-action-btn">
        <span class="material-symbols-rounded">play_arrow</span>
        Текст на кнопке
    </button>

    <div id="УНИКАЛЬНЫЙ-ID-КАРТОЧКИ" class="md-card elevated-card program-result-card">
        <h3 class="title-medium">
            <span class="material-symbols-rounded">analytics</span> Заголовок результатов
        </h3>
        <div id="УНИКАЛЬНЫЙ-ID-КОНТЕНТА" class="body-medium">
            </div>
    </div>

</div>

<script src="progs/имя_папки_программы/ui.js"></script>
3. Глобальные CSS-стандарты для форм (style.css)
Использовать эти классы для сохранения единого стиля Material Design 3:

.program-form-container — общая flex-обёртка для элементов формы (задаёт вертикальный шаг в 20px).

.program-label — стиль для подписей полей.

.program-input — адаптивное текстовое поле с мягким подсветом при фокусе.

.program-action-btn — кнопка действия темы (Filled Button).

.program-result-card — скрытый по умолчанию контейнер для вывода результатов.

4. Как вызывать программы в модальном окне
На любую плитку или элемент списка вешается универсальный JS-вызов:

HTML
onclick="openProgramModal('Заголовок окна для людей', 'имя_папки_программы')"
Система сама сходит в ui_render.php, заберёт разметку, оживит внутренние теги <script>, заблокирует фоновый скролл, а при закрытии — полностью сотрёт данные из памяти для предотвращения конфликтов ID.
